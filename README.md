# ERIK BARBER

Мобильный fashion-ориентированный сайт барбера с записью через `YCLIENTS`, личным кабинетом, отзывами, бонусной программой и магазином наград.

## Что уже сделано

- Главный экран без длинного скролла на телефоне.
- Полноэкранные панели для работ, дипломов, отзывов, кабинета и магазина.
- Мгновенное окно регистрации при первом входе.
- Локальный личный кабинет с бонусным балансом, историей событий и запросами мастеру.
- Блок демо-отзывов и форма отправки новых отзывов.
- Бонусный магазин:
  - `2000` бонусов — бесплатная стрижка.
  - `1500` бонусов — бесплатная стрижка бороды.
  - `1000` бонусов — рулетка с скрытыми шансами.
- Демо-кнопка в кабинете, которая имитирует начисление `500` бонусов после статуса визита.

## Важный архитектурный момент

`GitHub Pages` подходит для фронтенда, но не подходит для:

- хранения аккаунтов в общей базе,
- синхронизации между устройствами,
- реального начисления бонусов после смены статуса в `YCLIENTS`,
- автоматической отправки запросов мастеру.

Поэтому рекомендуемая схема такая:

1. Фронтенд держим в этом репозитории и публикуем на `GitHub Pages`.
2. Базу, регистрацию и отзывы подключаем через `Supabase`.
3. Прием событий из `YCLIENTS` делаем через `Supabase Edge Function` или `Cloudflare Worker`.

В этом проекте уже добавлены заготовки под `Supabase`.

## Что нужно сделать следующим этапом

1. Подключить `Supabase` и заменить локальный `localStorage` на облачную базу.
2. Связать клиента на сайте с клиентом в `YCLIENTS` по телефону.
3. Настроить обработчик статуса визита:
   - когда визит получает статус `Клиент пришел`,
   - создать событие начисления `+500`,
   - обновить баланс клиента,
   - записать историю в кабинет.
4. Настроить уведомление мастеру о списании бонусной награды.
